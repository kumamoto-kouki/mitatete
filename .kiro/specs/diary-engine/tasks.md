## Implementation Notes

- **着手前提（cross-spec）**: 本文生成は model-router の汎用生成エントリ `generate(system_prompt, messages) -> text` に依存する。model-router 実装でこのエントリを公開してから 2.1 に着手する（ユーザー決定：model-router 完成後）。
- **既存シーム再利用（変更なし）**: 強度導出は `src/principles.ts` の `calcDiaryIntensity`、履歴は storage の `read_history(date)`、保存は `save_diary(date, content)`（未承認時ローカルのみ・承認時同期は storage が担保）。当日 `YYYY-MM-DD` はフロントで生成。
- **テスト基盤**: 既存 vitest を使用（新規導入不要）。`diary-prompt.ts` の純関数と `diary.ts` のオーケストレーション（生成・履歴・保存をモック）を単体検証。日記パネルの実表示は `pnpm dev` で手動確認。
- **トリガー**: MVP はユーザー操作（生成ボタン）。自動生成は持たない（要件 1.3, 2.3）。
