# steering 索引（このディレクトリの地図）

`.kiro/steering/` は **毎セッション丸ごと読み込まれる**プロジェクトメモリ（CLAUDE.md の方針）。索引を置いても読み込み量は減らない（全部ロードされる）ため、**条件付き（特定パスに触れた時だけ要る）内容は eager な steering でなく lazy な `.claude/rules/` に置く**のが軽量化のレバー。

## steering（常時ロード）

| ファイル               | 内容                                                                                               | 所有             |
| ---------------------- | -------------------------------------------------------------------------------------------------- | ---------------- |
| `product.md`           | プロダクトの目的・原則                                                                             | PO／全員         |
| `tech.md`              | 技術スタック・制約                                                                                 | 全員             |
| `structure.md`         | ディレクトリ／コード構成                                                                           | 全員             |
| `orchestration.md`     | **中核モデル**：役割・レーン＋リーダー・委譲規律 A〜E・worktree 戦略・信用原則・振り返り・git 運用 | 👨🏼‍💼 神谷          |
| `operations.md`        | **運用統治**：報告系統・状態遷移＋全体整合・WIP 上限・ID 体系・開示の健全性                        | 👩🏼‍💼 橘            |
| `role-catalog.md`      | 役割カタログ＋投入計画（配役・候補・charter）                                                      | 👩🏼‍💼 橘            |
| `review-checklists.md` | レビュー受理観点（セキュリティ／パフォ／a11y／原則／状態／プロセス）                               | 🕵🏼‍♀️ 望月／🛡️ 守屋 |

## rules（パス連動 lazy ロード＝該当ファイルに触れた時だけ）

| ファイル                          | 発火パス       | 内容                                |
| --------------------------------- | -------------- | ----------------------------------- |
| `.claude/rules/frontend-state.md` | `src/**`       | フロント状態・Tauri 配線の判断基準  |
| `.claude/rules/rust-backend.md`   | `src-tauri/**` | Rust/Tauri バックエンドの判断基準   |
| `.claude/rules/design-review.md`  | CSS／design 系 | デザイン受理条件・/design-sync 手順 |

## 置き場所の原則（軽量化のレバー）

- **常に要る** → steering（上の表を肥大させない）
- **特定パスに触れた時だけ** → `.claude/rules/`（lazy・読み込み量を抑える）
- **参照・背景** → `docs/` ・ `.claude/reports/`（オンデマンド read）
