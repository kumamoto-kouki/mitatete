# 依頼：Mitatete のデザインシステムを作ってください

## プロダクト

Mitatete は、AI モデルを「見立て（mitate）」によって擬人化し、人間と AI の心理的距離を縮めるデスクトップアプリです（Tauri 製）。
ユーザーは AI モデル（Claude / GPT / Gemini 等）を、性格やビジュアルを持つ「キャラクター」として選び、対話します。

## 最重要の制約（絶対に守る原則8：AI開示）

- どの画面でも「これは AI である」ことを**隠さない／偽装しない**。擬人化はするが、人間のフリはさせない。
- 各会話画面に常設の「AI開示バナー（これは AI による応答です、の明示）」を必ず設計に含めてください。これは装飾で消してよい要素ではなく、**機能的に必須**の固定要素です。
- かわいい・親しみやすい表現でも、AI であることの誠実な明示と両立させてください。

## トーン & 性格

- 温かい・親しみやすい・やわらかい。擬人化だが誠実。
- ベースの配色（出発点。改善提案歓迎）：温かいベージュ/ブラウン系。
  - 背景 `#faf8f4` / カード `#fff` / 文字 `#2b2622` / 補助文字 `#8a817a` / 罫線 `#e6e0d8` / アクセント `#b08968`
  - 状態色：成功 `#2e7d52` / 注意 `#d9a441` / 危険 `#b00020` / 情報 `#3a6ea5`

## 画面構成（2 ウィンドウ）

1. **メイン窓**：チャット（入力欄・会話バブル・AI開示バナー）＋モデル選択パネル＋キャラクター選択パネル。
2. **キャラクター窓**：選択中キャラクターのビジュアル表示（立ち絵/アバター＋名前/状態）。

## 技術制約（実装可能性が受理条件）

- Tauri v2 ＋ **素の TypeScript ＋素の CSS**。重い UI フレームワーク（React 等）は使わない。
- 各コンポーネントは **プレーンな HTML/CSS で実装できる**こと。CSS 変数（デザイントークン）前提で。
- アクセシビリティ：**WCAG AA**（本文コントラスト 4.5:1・大きな文字/UI 3:1）。フォーカスリング可視。

## 作ってほしいもの（デザインシステム）

基盤トークンと、以下のコンポーネントを**それぞれ独立した HTML プレビュー**で。各プレビューに状態バリエーションを含める。

1. **デザイントークン**：カラー / タイポ（日本語含む。`system-ui, "Hiragino Kaku Gothic ProN", "Noto Sans JP"` を想定）/ スペーシング / 角丸 / 影。
2. **ボタン**：primary / secondary / ghost、各 hover・focus・disabled。
3. **チャットバブル**：ユーザー発話 / AI 発話。AI 発話には小さな「AI」ラベルを必ず添える。
4. **AI開示バナー**：会話画面上部の常設バナー（原則8）。控えめだが必ず視認できる。
5. **モデル選択パネル**：複数モデルをカードで選択。選択中の状態を明示。
6. **キャラクターカード / キャラクター表示**：アバター＋名前＋ひとこと性格＋選択状態。
7. **入力欄**：通常 / focus / disabled / 文字数や送信中（loading）状態。
8. **状態網羅の見本**：empty（会話なし）/ error（送信失敗）/ loading の表示。

## 出力形式（/design-sync 連携のため）

- **1 コンポーネント＝1 つの HTML プレビューファイル**に分けてください（後でローカルのコンポーネントライブラリへ 1 個ずつ同期します）。
- 各プレビュー先頭にカード見出しコメント（例：`<!-- @dsCard group="Components" -->`）を付けてください。
- 色・余白・タイポは直書きせず、トークン（CSS 変数）を参照する形に。
- エクスポート（HTML / ZIP）できる状態にしてください。

## 進め方

- まず**トークンと AI開示バナーとチャットバブル**から提案してください（このプロダクトの核のため）。
- 不明点（キャラクターの世界観・対象ユーザー・ダーク対応の要否など）があれば、作り始める前に質問してください。
