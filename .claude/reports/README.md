# reports/ — 実装後の振り返り

タスク・スペックの実装がひと区切りついたタイミングで、軽量な振り返りを書く。

## 書くタイミング
- 1つのspec(またはタスクのまとまり)の実装が完了した直後
- 想定外の手戻りや、何度も同じ判断をやり直した箇所があったとき

## 書く内容(短くてよい)
- **何を実装したか**: 1-2行
- **詰まった点・繰り返した判断**: そこにどんな判断基準があれば早く進めたか
- **`.claude/rules/` 化の候補**: 今後も発生しうるパス連動の判断基準があれば
- **スキル化の候補**: 繰り返し手順があれば `.claude/skills/` 化を検討

## ファイル名
`YYYY-MM-DD-<feature-or-topic>.md`

## 手入れ(Garbage Collection)
- ルール/スキルに昇格した内容は、reportに昇格先へのリンクを残してよい
- 古い報告は削除せず、ルール化の判断材料として残す
