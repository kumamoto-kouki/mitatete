# rules/ — パス連動の規約ファイル

このディレクトリには、特定のパスに触れる際に自動的に読み込まれるべき「判断基準」を置く。

## 書くタイミング
- ルールは実装が先、ルール化は後。コードがまだ存在しない/パターンが1回しかない段階では書かない
- 同じ判断を2回目以降下す場面が来たら、その判断基準を切り出す
- `.claude/reports/` の振り返りで「このルールは切り出す価値がある」と判断したものを正式化する

## 書き方
- frontmatterに `paths` (glob) を書き、対象ファイルに触れたときだけ読み込まれるようにする
- 本文は **手順(How)ではなく判断基準(Why)** を書く。手順は陳腐化するが、判断基準は応用できる
- 短く。1ルール1テーマ

### テンプレート
```markdown
---
paths:
  - path/to/target/**
---
## <ルール名>
- <判断基準1>(なぜそうするか)
- <判断基準2>
```

## 手入れ(Garbage Collection)
- 対応するコード/パターンが消えたルールは削除する
- `.claude/reports/` での振り返り時に、使われていないルールがないか確認する
